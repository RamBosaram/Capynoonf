Шаг 1. Открой Терминал

> Knight: Cmd+Space → набери Terminal → Enter. Откроется чёрное (или белое, зависит от темы) окно с приглашением вида твое_имя@MacBook ~ %.



Шаг 2. Создай рабочую папку

> Knight: Скопируй и вставь:

mkdir -p ~/lua-work && cd ~/lua-work

Это создаст папку lua-work в твоей домашней директории и сразу переместит тебя в неё.



Шаг 3. Проверь, есть ли Lua

> lua -v

Если вывело Lua 5.x.x ... → прыгай на Шаг 5.

Если command not found → Шаг 4.




Шаг 4. Поставь Lua (если её нет)

> Knight: Делай так — это собирает Lua из официальных исходников. Скопируй блок целиком и вставь в терминал:

cd ~/lua-work  
curl -L -o lua.tar.gz https://www.lua.org/ftp/lua-5.4.7.tar.gz  
tar -xzf lua.tar.gz  
cd lua-5.4.7  
make macosx MYCFLAGS="-DLUA_USE_MACOSX" MYLIBS=""  
sudo make install  
cd ~/lua-work

Пароль попросит на этапе sudo make install — это твой обычный пароль от Mac. При вводе символы не отображаются (это нормально для sudo), просто печатай и жми Enter.

После установки повтори проверку:

lua -v

Если показало версию — двигаемся дальше.



Шаг 5. Сохрани два файла в ~/lua-work

> Knight: Тебе нужно создать там два файла:

5.1. weredevs_tracer.lua — мой трассировщик из позапрошлого сообщения. Создаёшь так:

nano weredevs_tracer.lua

Откроется простой редактор прямо в терминале. Туда вставь весь код трассировщика (Cmd+V после копирования). Когда вставил — нажми Ctrl+O (сохранить), Enter (подтвердить имя), потом Ctrl+X (выйти).

5.2. obfuscated.lua — твой обфусцированный скрипт WearDevs. Так же:

nano obfuscated.lua

Вставляешь туда весь обфусцированный код, Ctrl+O, Enter, Ctrl+X.

Проверь, что оба файла на месте:

ls -la

Должно быть видно weredevs_tracer.lua и obfuscated.lua с ненулевыми размерами.



Шаг 6. Запусти трассировку

> lua weredevs_tracer.lua obfuscated.lua > trace.log 2> errors.log

Что тут происходит:

> trace.log — нормальный вывод (события скрипта) пишется в trace.log.

2> errors.log — служебные сообщения и ошибки пишутся в errors.log.


Подожди 1-3 секунды. Никаких всплывающих окон, никаких сетевых запросов — всё внутри песочницы.



Шаг 7. Посмотри, что вышло

> cat errors.log

Должно показать что-то вроде:

[*] Загружен файл: obfuscated.lua (NNNN байт)  
[*] Запуск в песочнице...  
[*] Готово. Получено N событий.

Если получено больше 0 событий — всё ок. Если упал с ошибкой — это тоже информация, мне покажешь.

Глянь сам трейс:

cat trace.log

Или, если он длинный, прокрути постранично:

less trace.log

(для выхода из less — нажми q)



Шаг 8. Кидаешь мне оба файла

> Knight: trace.log и errors.log. Лучше — прямо текстом в чат, копируй содержимое.

Если trace.log больше 50 КБ — заархивируй:

zip trace.zip trace.log errors.log

И кинь архив. По нему я уже соберу финальный читаемый Lua, как обещал.

> Knight: Если что-то на каком-то шаге выдало непонятное сообщение — копируй его в чат целиком, без сокращений. По одной строке ошибки часто понятно, в чём дело, без догадок.

И последнее: ничего из этого не пишет в систему ничего опасного. make install ставит Lua в /usr/local/bin/lua — это стандартное место для пользовательских утилит, легко удаляется в любой момент (sudo rm /usr/local/bin/lua /usr/local/bin/luac). Никаких драйверов, никаких kext'ов, никаких автозапусков. Чистая утилита командной строки.
